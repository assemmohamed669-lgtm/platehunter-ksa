# 🎙️ توثيق كامل لميزة التسجيل الصوتي — PlateHunter KSA

هذا الملف يشرح **كل حاجة** خاصة بصفحة التسجيل: من أول ما تضغط زر المايك،
لحد ما تطلع اللوحات في ملف إكسيل — التعرف الصوتي، الـ GPS، تحويل الصوت للوحة،
التخزين، والتصدير. مع أسماء الملفات والدوال والأدوات المستخدمة.

> آخر تحديث: بعد إلغاء شاشة المراجعة الإجبارية (حفظ فوري + تصحيح لاحق في الجدول)،
> إضافة تعلّم أخطاء السمع (letter-confusion)، علامة "مشكوك فيه"، واختيار أفضل
> تخمين صوتي بالثقة (confidence).

سحابية**. هو كود محلي 100% في `lib/plateParser.ts` بيشتغل أوفلاين. الجهاز بيحوّل

**ملف `app/api/analyze-plate/route.ts`** موجود في المشروع وبيستخدم Anthropic API
(مدفوع)، **لكنه مش متصل بأي زرار فعلي** في صفحة التسجيل — كود ميت من محاولة قديمة.
التفريغ الفعلي كله محلي عن طريق `plateParser.ts`.

## 2. رحلة التسجيل خطوة بخطوة (محدّثة — بدون مراجعة إجبارية)

```
1. المستخدم يضغط زر المايك (🎙️)
       ↓
2. الجهاز يبدأ يسمع ويحوّل الكلام لنص (Speech-to-Text)
   - لكل جملة نهائية (isFinal)، بياخد كذا تخمين (alternatives) من الجهاز
     ويختار أفضلهم بـ pickBestHypothesis (انظر قسم 3.ج)
       ↓
3. المستخدم يضغط "وقف"
       ↓
4. النص الكامل يتخزّن مؤقتاً (pendingTranscript) ويظهر في لوحة "جاهز للتفريغ"
       ↓
5. المستخدم يضغط "احفظ وصدّر الإكسيل"
       ↓
6. extractMultiplePlates(النص) → يطلّع مصفوفة لوحات
   (لو فشلت، fallback على parsePlateFromTranscript للوحة واحدة)
       ↓
7. كل لوحة تتصحح تلقائياً بأنماط متعلَّمة من أخطاء سمع سابقة
   (applyLetterConfusions) — انظر قسم 7
       ↓
8. كل لوحة تتحفظ فورًا في IndexedDB (بدون أي شاشة مراجعة/تعديل تمنع الحفظ)
   + يتجاب لها الحي/الشارع من الـ GPS + تتفحص مقابل ملف التشييك
       ↓
9. يتفتح ملف إكسيل فوري بكل لوحات الجلسة دي
```

**ليه اتشالت شاشة المراجعة الإجبارية؟** المستخدم بيسجّل وهو ماشي في الشارع —
مش عملي إنه يوقف يراجع/يصحح كل لوحة قبل ما تتحفظ. الحل: حفظ فوري بأفضل تخمين
ممكن، والتصحيح (لو احتاج) بيحصل بعدين في أي وقت بضغطة على اللوحة في الجدول
(انظر قسم 8).

## 3. التعرف على الكلام (Speech-to-Text)

في نظامين حسب مكان التشغيل:

### أ) على الموبايل (APK) — Capacitor Speech Recognition
المكتبة: `@capacitor-community/speech-recognition`

```ts
const { SpeechRecognition } = await import("@capacitor-community/speech-recognition");
await SpeechRecognition.requestPermissions();

// حلقة إعادة تشغيل تلقائي — تفضل تسمع لحد ما المستخدم يوقف
while (isRecordingRef.current) {
  try {
    const result = await SpeechRecognition.start({
      language: "ar-SA",
      maxResults: 5,          // كذا تخمين، مش تخمين واحد بس
      partialResults: false,
      popup: false,
    });
    const text = pickBestHypothesis(result?.matches ?? []);
    if (text) finalTranscriptRef.current += " " + text;
  } catch (err) {
    // صمت / لا يوجد كلام → أعِد المحاولة، لا توقف الجلسة
    if (!isRecordingRef.current) break;
    await new Promise((r) => setTimeout(r, 300));
  }
}
```

**ملاحظة:** المحرك بيرمي خطأ عند أي صمت قصير بين اللوحات. الكود بيعيد المحاولة
تلقائياً ومايوقفش إلا لما تضغط "وقف" بنفسك.

### ب) على المتصفح (ويب) — Web Speech API
```ts
const SR = window.SpeechRecognition ?? window.webkitSpeechRecognition;
const recognition = new SR();
recognition.lang = "ar-SA";
recognition.maxAlternatives = 5;   // كذا تخمين لكل جملة
recognition.onresult = (event) => {
  // لكل نتيجة نهائية: يجمع كل البدائل + درجة ثقة كل واحد (confidence)
  // ويختار أفضلهم بـ pickBestHypothesis(alts, confs)
};
recognition.onend = () => { if (isRecordingRef.current) recognition.start(); }; // إعادة تشغيل
```

### ج) اختيار أفضل تخمين — `pickBestHypothesis` (في `plateParser.ts`)

الجهاز بيرجّع كذا احتمال لنفس الجملة (alternatives). الكود بيختار الأنضف منهم:

```ts
pickBestHypothesis(candidates: string[], confidences?: number[]): string
```

- **`plateContentScore(text)`**: بيدي نقطة لكل كلمة — حروف نضيفة (دال، صاد...) أو
  مجموعات أرقام تاخد نقط موجبة، كلمات طويلة/غريبة (زي "راقوف" أو "السلام عليكم")
  تاخد نقطة سالبة.
- كل تخمين (alternative) بياخد `plateContentScore` + درجة ثقة الجهاز نفسه
  (`confidence`, لو متاحة). الأعلى مجموع هو اللي يتختار.
- **درجة الثقة (confidence) متاحة بس على متصفح الويب** (Chrome/Edge) — الـ
  Capacitor plugin على الأندرويد ما بيرجعهاش، فهناك الاختيار بيعتمد على
  `plateContentScore` بس.
- الثقة بتشتغل كـ *مرجّح صغير (tiebreaker)* بس — تقدر تحسم تعادل قريب، مش تقلب
  قرار واضح المصدره من `plateContentScore`.

### ⚠️ حدود جودة التعرف الصوتي (مهم)
- محرك أندرويد (`ar-SA`) **مش مصمم لتهجئة الحروف**. لما تقول حروف لوحة متهجّاة،
  بيحاول يحوّلها لكلمات حقيقية:
  - "راء قاف" → قد يسمعها **"راقوف"** (كلمة واحدة)
  - "بطس" → قد يسمعها "ب **ت** س" (ت مش حرف لوحة سعودية)
  - أحياناً **يبلع أرقام** أو يقلبها (6469 → "6 9").
- **نصيحة عملية:** انطق كل حرف باسمه الكامل وبوقفة صغيرة: "راء ... قاف ... سين".
  المعالج بيفهم أسماء الحروف مظبوط، فده بيحسّن النتيجة كتير مجاناً.
- **الحل الأقوى (مؤجَّل):** تفريغ سحابي (Whisper/Groq) — أدق بكتير لكن يحتاج
  حساب API. مش مفعّل حالياً.

### ملاحظة: تعارض المايكروفون + تسجيل الصوت الخام
جهاز واحد بس ممكن يستخدم المايك في نفس الوقت — فمفيش تسجيل صوت خام (ملف صوتي
تسمعه/تشاركه) شغال حالياً بالتوازي مع التعرف على الكلام. **تسجيل الصوت الخام محتاج
مكتبة Capacitor جديدة غير مثبتة حالياً + بناء APK جديد** (مش مجرد `git push`) —
مؤجّل لحد ما يتقرر تنفيذه.

## 4. تحديد الموقع (GPS)

الملف: `lib/gps.ts` — كائن مفرد `gpsService` (singleton).

### كيف يشتغل
- **موبايل:** `@capacitor/geolocation` مع `watchPosition` (متابعة مستمرة).
- **ويب:** `navigator.geolocation.watchPosition`.
- `enableHighAccuracy: true` — دقة عالية.
- يخزّن آخر إحداثيات في `lastCoords`، وينبّه المشتركين كل 5 ثواني.

### الدوال الأساسية
```ts
gpsService.startTracking();          // يبدأ المتابعة (عند فتح الصفحة)
gpsService.subscribe((coords)=>{});  // يشترك في التحديثات
gpsService.getLastCoords();          // آخر إحداثيات معروفة (فوري)
gpsService.pinCurrentLocation();     // يجيب الموقع الحالي مرة واحدة (زر الدبوس اليدوي)
gpsService.stopTracking();           // يوقف المتابعة (عند مغادرة الصفحة)
```

### دوال مساعدة (exported)
```ts
toMapsLink(lat, lng)              // → "https://www.google.com/maps?q=LAT,LNG"
extractLatLngFromMapsLink(url)    // العكس: يطلّع {lat,lng} من رابط خرايط
haversineKm(lat1,lng1,lat2,lng2)  // المسافة بين نقطتين بالكيلومتر
```

### وقت الالتقاط
عند بداية التسجيل، يتم "تجميد" الإحداثيات الحالية:
```ts
gpsAtRecordRef.current = gpsService.getLastCoords();
```
فكل لوحة في التسجيل بتاخد نفس إحداثيات لحظة بداية التسجيل.

## 6. تحويل الكلام للوحة — المعالج (أهم جزء)

الملف: `lib/plateParser.ts`
الدالة: `extractMultiplePlates(transcript: string): MultiPlateResult[]`

> كود محلي بالكامل — **لا AI، لا إنترنت**. مجرد regex + قواميس + خوارزمية.

### القاعدة الأساسية
كل لوحة سعودية = **1-3 حروف** + **4 أرقام** (تتكمّل بأصفار لو أقل).
الحروف الصحيحة فقط: `ا ب ح د ر س ص ط ع ق ك ل م ن هـ و ي`

### مراحل المعالجة (Pipeline)

```
1. تنظيف التشكيل (removeDiacritics) — يشيل الحركات فقط، مش الحروف
2. توحيد الألف: أ إ آ → ا
3. توحيد الهاء: ه → هـ
4. ى → ي
5. قاموس أسماء الحروف (LETTER_NAMES): دال→د، صاد→ص، لام→ل، سين→س ...
6. الدمج الصوتي (PHONETIC_MERGES): احلام→ا ح ل ...
7. الأرقام المنطوقة (SPOKEN_NUMBERS): خمسة→5، تلاتين→30، ألفين→2000 ...
8. توحيد الأرقام العربية: ٥→5
       ↓
9. تقطيع لـ "atoms" (ذرّات): كل كلمة تتصنّف:
   L = حرف لوحة | D = رقم | V = نوع سيارة | N = ملاحظة
       ↓
10. تجميع الأرقام في مجموعات (كل مجموعة 4 أرقام = لوحة واحدة)
       ↓
11. لكل مجموعة أرقام: امسح للخلف واجمع أقرب ≤3 حروف نضيفة (L atoms) = لوحة.
    لو مفيش حروف نضيفة قدام، حاول تسحب حروف من كلمة متلخبطة جنبها (fallback) —
    وده يخلّي اللوحة "مشكوك فيها" (uncertain، انظر قسم 6.ب)
       ↓
12. باقي الكلمات → ملاحظات لأقرب لوحة، وأنواع السيارات → خانة النوع
```

### مثال حي (من تسجيل حقيقي)
```
تقول:            "دال لام لام تسعة ستة سبعة تسعة"
بعد التطبيع:      "د ل ل 9 6 7 9"
النتيجة:          دلل9679 ✅
```

### 6.أ — قواميس مهمة داخل الملف
- `LETTER_NAMES` — أسماء الحروف بالفصحى والعامية المصرية (دال، صاد، حاء، حا...).
- `EGYPTIAN_LETTERS` — النطق المصري القصير (حه→ح، دال→د...).
- `PHONETIC_MERGES` — كلمات بتتقال ملزوقة (احلام→ا ح ل).
- `SPOKEN_NUMBERS` — كل الأرقام المنطوقة من 0 لـ 9000 + المركّبة (خمسة وعشرين→25).
- `VEHICLE_TYPES` — ونيت، فان، دباب، شاحنة، باص، صالون، بيكاب، تاكسي، كروزر، باترول، نقليات، مفحوطة.
- `NOTE_KEYWORDS` — كلمات اتجاه/مكان (يمين، جراج، مركونة...) بتتحط في الملاحظات
  دايماً، حتى لو حروفها شكلها حروف لوحة صحيحة.
- `VALID_AR_LETTERS` — الـ 17 حرف المسموح بيهم على اللوحات السعودية.

### 6.ب — علامة "مشكوك فيه" (`uncertain`)

كل من `MultiPlateResult` و`ParseResult` فيهم حقل اختياري `uncertain?: boolean`.
بيتحط `true` لما:
- الحروف اتسحبت من كلمة متلخبطة (fallback) مش من حروف نضيفة مجاورة للرقم مباشرة.
- مفيش حروف اتلاقت خالص قبل أو بعد الأرقام.
- في `parsePlateFromTranscript`: استخدم المسار الاحتياطي (regex fallback أو
  char-extraction fallback) بدل المسح الأساسي النضيف.

الحقل ده بيتوصّل لـ `RecordingEntry.uncertain` في IndexedDB، وبيظهر كعلامة ⚠️
جنب اللوحة في الجدول (انظر قسم 8).

### 6.ج — دالة اللوحة الواحدة
`parsePlateFromTranscript()` — نفس المنطق بس للوحة واحدة (بتستخدم كـ fallback
لو `extractMultiplePlates` رجّع مصفوفة فاضية).

### الاختبارات
`__tests__/plateParser.test.ts` — **185 اختبار** يغطّوا كل الحالات
(لوحة واحدة، متعددة، أسماء حروف، ملاحظات، أنواع سيارات، أرقام منطوقة، حالات
مشوّهة، اختيار أفضل تخمين، تعلّم أخطاء السمع، علامة "مشكوك فيه").
```bash
npx vitest run    # تشغيل كل الاختبارات
```

## 7. تعلّم أخطاء السمع — Letter-Confusion Learning

كل جهاز/مايك ليه أنماط أخطاء سمع ثابتة (مثلاً بيسمع "ص" كـ"س" دايماً). النظام
بيتعلّم من تصحيحاتك ويطبّقها تلقائياً في المرة الجاية، **قبل** ما تشوف اللوحة.

### الدوال (في `plateParser.ts`)
```ts
diffLetterCorrections(extractedPlate, correctedPlate)  // فرق حرف-بحرف بين
  // النص المسموع والنص المصحّح — بس لو الأرقام وعدد الحروف متطابقين (يعني
  // احنا متأكدين إنها نفس اللوحة، مش لوحة تانية خالص)

recordLetterCorrections(map, extractedPlate, correctedPlate)  // يسجّل كل فرق
  // في خريطة (heard letter → corrected letter → عدد مرات)

applyLetterConfusions(plate, map, minCount=3, minDominance=0.7)  // يصحح لوحة
  // جديدة تلقائياً — بس لو نفس التصحيح اتكرر 3 مرات على الأقل وكان غالب
  // (70%+ من كل التصحيحات المسجّلة لنفس الحرف)، عشان تصحيح نادر أو متردد
  // مايتطبقش غلط

serializeLetterConfusions(map) / deserializeLetterConfusions(obj)  // تحويل
  // الخريطة لـ/من شكل JSON عشان تتخزن في localStorage
```

### أين بيتخزن؟
`localStorage`, مفتاح `ph:registration:letterConfusions` — محلي على الجهاز، مش
بيتزامن مع السيرفر (كل جهاز بيتعلم أنماط أخطائه بنفسه).

### متى بيتعلّم؟
كل مرة تعدّل لوحة في الجدول (`handlePlateEdit` في `page.tsx`) — بشرط إن اللوحة
دي كانت من **تسجيل صوتي** (مش إدخال يدوي، لأن الكتابة اليدوية مفيهاش "سمع" أصلاً
تتعلم منه). التصحيح بيتقارن مع `originalPlate` (القيمة الخام قبل أي تصحيح تلقائي).

## 8. صفحة التسجيل — الحفظ والتعديل (بدون مراجعة إجبارية)

الملف: `app/(app)/registration/page.tsx`

### الحفظ الفوري
زرار "احفظ وصدّر الإكسيل" بيعمل (`handleTranscribeAndSave`):
1. `extractPlates(transcript)` — استخراج + تصحيح تلقائي (letter-confusion) +
   علامة `uncertain` (parser + هل اتصحح تلقائي).
2. `savePlateList(...)` — حفظ فوري في IndexedDB، بدون أي انتظار لمراجعة يدوية.
3. تصدير إكسيل فوري بلوحات الجلسة دي.

### التعديل بعد الحفظ (`handlePlateEdit`)
في `RecordingsTable.tsx`، كل لوحة قابلة للضغط (فيها أيقونة ✏️) — تضغط عليها
تتحول لخانة نص، تعدّل وتضغط Enter أو تخرج منها (blur):
1. يتعلّم من الفرق (لو مش إدخال يدوي) — `recordLetterCorrections`.
2. يحدّث `idb.updatePlate(localId, plate)` — وده بيشيل علامة `uncertain` تلقائياً
   لأن حد بشري بقى شافها وأكدها.
3. يعيد فحص اللوحة المصحّحة مقابل ملف التشييك (`checkPlateMatch`).

### علامة "مشكوك فيه" ⚠️
لوحة اتحفظت بثقة واطية (fallback أو تصحيح تلقائي) بتظهر بعلامة تحذير كهرمانية
جنبها في الجدول — عشان لو حبيت تراجع، عينك تروح على اللوحات دي بس مش كل الجدول.

## 9. التخزين المحلي (IndexedDB)

الملف: `lib/idb.ts`

### شكل السجل الواحد (RecordingEntry)
```ts
interface RecordingEntry {
  localId: string;           // uuid محلي
  agentId: string;           // معرّف مستخدم Supabase
  plate: string;             // اللوحة بدون فراغات مثل أبح1234
  originalPlate?: string;    // القيمة الخام قبل أي تصحيح — لتعلّم أخطاء السمع
  uncertain?: boolean;       // الاستخراج مش متأكد منه — يستاهل نظرة
  vehicleType?: string;      // ونيت / فان / دباب ...
  lat?: number;
  lng?: number;
  street?: string;
  district?: string;
  recordedAt: string;        // ISO timestamp
  audioBlobBase64?: string;  // غير مستخدم حالياً — كود قديم من محاولة سابقة
  mapsLink?: string;
  notes?: string;
  recorderName?: string;
  isManual?: boolean;        // true = إدخال يدوي، false = تسجيل صوتي
  synced: boolean;           // اتزامن مع السيرفر؟
}
```

### الدوال
```ts
saveRecording(entry)            // يحفظ/يحدّث سجل
getAllRecordings(agentId)       // كل السجلات (مرتبة بالأحدث)
deleteRecording(localId)
updateNotes(localId, notes)
updatePlate(localId, plate)     // يعدّل اللوحة + يشيل علامة uncertain تلقائياً
updateGeodata(localId, street, district)
```

## 10. تصدير الإكسيل

الملف: `lib/excel.ts` (بيستخدم مكتبة **xlsx / SheetJS**)

### ترتيب الأعمدة (من `buildRows` في صفحة التسجيل)
```
رقم اللوحة | نوع السيارة | الشارع | الحي | تاريخ التسجيل | GPS | ملاحظات
```
- التاريخ يتنسّق كـ `DD-MM-YYYY HH:mm` (مش ISO خام).
- خلايا الـ GPS بتبقى روابط خرايط قابلة للضغط.

### الدوال
```ts
buildExcelBlob(rows, sheetName)          // يبني ملف إكسيل في الذاكرة
openExcelBlob(blob, filename)            // يفتح الملف (موبايل: FileOpener، ويب: تنزيل)
shareExcelBlob(blob, filename, title)    // يشارك عبر قائمة المشاركة (واتساب...)
```

### اسم الملف
قابل للتعديل من خانة "اسم ملف الإكسيل"، افتراضياً `اكسيل-DD-MM-YYYY`.
القيمة بتتحفظ في `localStorage`.

## 11. الخانات القابلة للتعديل (فوق زر التسجيل)

٣ خانات، كل واحدة معاها زر مسح (✕)، وكلها بتتحفظ في `localStorage`:

| الخانة | المفتاح | الوظيفة |

| اسم ملف الإكسيل | `ph:registration:excelName` | اسم ملف التصدير |

## 12. لوحة التشخيص (Debug Panel)

في أسفل الصفحة لوحة قابلة للطي بتوريك مراحل التحويل:
- **0 — SR Status:** حالة محرك التعرف الصوتي.
- **1 — Raw Transcript:** النص الخام (اللي الجهاز سمعه) — **أهم حاجة للتشخيص**.
- **2 — Original Transcript:** النص الأصلي من المحرك.
- **3 — Normalized Transcript:** بعد التطبيع.
- **4 — Plate Parsing Result:** اللوحات المستخرجة.
- **5 — Vehicle Type Result:** أنواع السيارات.
- **6 — Notes Result:** الملاحظات المتبقية.

> لو التفريغ طلع غلط، افتح "1 — Raw Transcript" وابعته — بيوضّح إذا كانت المشكلة
> في السمع (المحرك) ولا في المعالجة (الكود).

## 14. الملخص التنفيذي (أهم النقاط)

1. **التفريغ محلي بالكامل** — مفيش AI أو سحابة، مجرد كود ذكي في `plateParser.ts`.
2. **الجهاز بيسمع ويحوّل لنص**، ويختار أفضل تخمين من كذا احتمال (`pickBestHypothesis`).
3. **مفيش شاشة مراجعة إجبارية** — اللوحة بتتحفظ فورًا بأفضل تخمين + تصحيح تلقائي.
4. **تصحيح لاحق اختياري** — دوس على أي لوحة في الجدول وصححها وقت ما يريحك.
5. **النظام بيتعلّم من تصحيحاتك** (letter-confusion) — كل ما تصحح نفس الغلط
   يتكرر، يبدأ يصححه لوحده.
6. **علامة ⚠️** بتظهر على اللوحات المشكوك فيها بس — عشان تراجع القليل مش الكل.
7. **الـ GPS** بيشتغل بمكتبة Capacitor على الموبايل، ونتيجته بتتحوّل لعنوان عربي
   عبر OpenStreetMap المجاني.
8. **عنق الزجاجة** = جودة سمع الجهاز للحروف المتهجّاة، مش الكود. النصيحة: انطق
   الحروف بأسماء واضحة وبوقفات.
9. **185 اختبار** بيحموا منطق التحويل والتعلّم من أي كسر مستقبلي.
10. **تسجيل الصوت الخام (audio) مؤجّل** — يحتاج مكتبة Capacitor جديدة + بناء APK
    جديد، مش مجرد push.
