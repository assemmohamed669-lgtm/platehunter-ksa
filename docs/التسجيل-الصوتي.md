# 🎙️ توثيق كامل لميزة التسجيل الصوتي — PlateHunter KSA

هذا الملف يشرح **كل حاجة** خاصة بصفحة التسجيل: من أول ما تضغط زر المايك،
لحد ما تطلع اللوحات في ملف إكسيل — التعرف الصوتي، الـ GPS، تحويل الصوت للوحة،
التخزين، والتصدير. مع أسماء الملفات والدوال والأدوات المستخدمة.

> آخر تحديث: بعد إعادة كتابة `extractMultiplePlates` (نظام الـ atoms) + إصلاح
> باگ التشكيل + جعل التفريغ يدوي بزر "ابدأ التفريغ".

---

## 1. نظرة عامة سريعة

| الجزء | الأداة / التقنية | الملف |
|------|------------------|-------|
| صفحة التسجيل (الواجهة) | React + Next.js (App Router) | `app/(app)/registration/page.tsx` |
| التعرف على الكلام (متصفح) | **Web Speech API** (`webkitSpeechRecognition`) | نفس الملف |
| التعرف على الكلام (موبايل APK) | **@capacitor-community/speech-recognition** | نفس الملف |
| تحديد الموقع | **@capacitor/geolocation** (موبايل) + `navigator.geolocation` (ويب) | `lib/gps.ts` |
| تحويل الإحداثيات لعنوان | **OpenStreetMap Nominatim** (مجاني، بدون مفتاح) | `lib/geocoding.ts` |
| تحويل الكلام للوحة (المعالج) | كود محلي بالكامل (regex + قواميس) | `lib/plateParser.ts` |
| التخزين المحلي | **IndexedDB** | `lib/idb.ts` |
| تصدير الإكسيل | **xlsx (SheetJS)** | `lib/excel.ts` |
| المزامنة السحابية | **Supabase** | `lib/sync.ts` |

**نقطة مهمة جداً:** التفريغ (تحويل الكلام للوحة) **لا يستخدم أي ذكاء اصطناعي أو خدمة
سحابية**. هو كود محلي 100% في `lib/plateParser.ts` بيشتغل أوفلاين. الجهاز بيحوّل
الصوت لنص (Speech-to-Text)، وبعدها الكود المحلي بيحلل النص ويطلّع اللوحات.

## 2. رحلة التسجيل خطوة بخطوة

```
1. المستخدم يضغط زر المايك (🎙️)
       ↓
2. الجهاز يبدأ يسمع ويحوّل الكلام لنص (Speech-to-Text)
   - يفضل مفتوح ويسجّل كل الكلام لحد ما تضغط "وقف"
       ↓
3. المستخدم يضغط "وقف"
       ↓
4. النص الخام (RAW TRANSCRIPT) يتخزّن مؤقتاً في الذاكرة (pendingTranscript)
   - لسه مفيش أي لوحة اتحفظت!
       ↓
5. تظهر لوحة "جاهز للتفريغ" فيها النص + زر "ابدأ التفريغ"
       ↓
6. المستخدم يضغط "ابدأ التفريغ"
       ↓
7. extractMultiplePlates(النص) → يطلّع مصفوفة لوحات
       ↓
8. كل لوحة تتحفظ في IndexedDB + يتجاب لها الحي/الشارع من الـ GPS
       ↓
9. يتفتح ملف إكسيل فوري بكل لوحات الجلسة دي
```

**ليه التفريغ بقى يدوي؟** عشان المستخدم يقدر يسمع/يراجع قبل الحفظ، ومايتحفظش
أي شيء غلط أوتوماتيك.

## 3. التعرف على الكلام (Speech-to-Text)

في نظامين حسب مكان التشغيل:

### أ) على الموبايل (APK) — Capacitor Speech Recognition
المكتبة: `@capacitor-community/speech-recognition`

```ts
const { SpeechRecognition } = await import("@capacitor-community/speech-recognition");
await SpeechRecognition.requestPermissions();

// حلقة إعادة تشغيل تلقائي — تفضل تسمع لحد ما المستخدم يوقف
while (isRecordingRef.current) {
  try {
    const result = await SpeechRecognition.start({
      language: "ar-SA",
      maxResults: 1,
      partialResults: false,
      popup: false,
    });
    const text = result?.matches?.[0] ?? "";
    if (text) finalTranscriptRef.current += " " + text;
  } catch (err) {
    // صمت / لا يوجد كلام → أعِد المحاولة، لا توقف الجلسة
    if (!isRecordingRef.current) break;
    await new Promise((r) => setTimeout(r, 300));
  }
}
```

**نقطة مهمة اتصلحت:** المحرك بيرمي خطأ عند أي صمت قصير بين اللوحات. الكود القديم
كان بيوقف التسجيل عند أول خطأ. دلوقتي بيعيد المحاولة تلقائياً ومايوقفش إلا لما
تضغط "وقف" بنفسك.

### ب) على المتصفح (ويب) — Web Speech API
```ts
const SR = window.SpeechRecognition ?? window.webkitSpeechRecognition;
const recognition = new SR();
recognition.lang = "ar-SA";
recognition.continuous = false;
recognition.interimResults = true;
recognition.onresult = (event) => { /* يجمع النص */ };
recognition.onend = () => { if (isRecordingRef.current) recognition.start(); }; // إعادة تشغيل
```

### ⚠️ حدود جودة التعرف الصوتي (مهم)
- محرك أندرويد (`ar-SA`) **مش مصمم لتهجئة الحروف**. لما تقول حروف لوحة متهجّاة،
  بيحاول يحوّلها لكلمات حقيقية:
  - "راء قاف" → قد يسمعها **"راقوف"** (كلمة واحدة)
  - "بطس" → قد يسمعها "ب **ت** س" (ت مش حرف لوحة سعودية)
  - أحياناً **يبلع أرقام** أو يقلبها (6469 → "6 9").
- **نصيحة عملية:** انطق كل حرف باسمه الكامل وبوقفة صغيرة: "راء ... قاف ... سين".
  المعالج بيفهم أسماء الحروف مظبوط، فده بيحسّن النتيجة كتير مجاناً.
- **الحل الأقوى (مؤجَّل):** تفريغ سحابي (Whisper/Groq) — أدق بكتير لكن يحتاج
  حساب API. مش مفعّل حالياً.

### ملاحظة: تعارض المايكروفون
جرّبنا تسجيل الصوت الحقيقي (ملف صوتي تسمعه/تشاركه) بالتوازي مع التعرف على الكلام،
لكن **جهازك بيسمح لمصدر واحد بس يستخدم المايك في نفس الوقت** — فتسجيل الصوت كان
بيحرم التعرف من السمع. القرار: أولوية للتعرف (استخراج اللوحة أهم)، فتسجيل الصوت
الحقيقي **اتشال** حالياً.

## 4. تحديد الموقع (GPS)

الملف: `lib/gps.ts` — كائن مفرد `gpsService` (singleton).

### كيف يشتغل
- **موبايل:** `@capacitor/geolocation` مع `watchPosition` (متابعة مستمرة).
- **ويب:** `navigator.geolocation.watchPosition`.
- `enableHighAccuracy: true` — دقة عالية.
- يخزّن آخر إحداثيات في `lastCoords`، وينبّه المشتركين كل 5 ثواني.

### الدوال الأساسية
```ts
gpsService.startTracking();          // يبدأ المتابعة (عند فتح الصفحة)
gpsService.subscribe((coords)=>{});  // يشترك في التحديثات
gpsService.getLastCoords();          // آخر إحداثيات معروفة (فوري)
gpsService.pinCurrentLocation();     // يجيب الموقع الحالي مرة واحدة (زر الدبوس اليدوي)
gpsService.stopTracking();           // يوقف المتابعة (عند مغادرة الصفحة)
```

### دوال مساعدة (exported)
```ts
toMapsLink(lat, lng)              // → "https://www.google.com/maps?q=LAT,LNG"
extractLatLngFromMapsLink(url)    // العكس: يطلّع {lat,lng} من رابط خرايط
haversineKm(lat1,lng1,lat2,lng2)  // المسافة بين نقطتين بالكيلومتر
```

### وقت الالتقاط
عند بداية التسجيل، يتم "تجميد" الإحداثيات الحالية:
```ts
gpsAtRecordRef.current = gpsService.getLastCoords();
```
فكل لوحة في التسجيل بتاخد نفس إحداثيات لحظة بداية التسجيل.

## 5. تحويل الإحداثيات لعنوان (Reverse Geocoding)

الملف: `lib/geocoding.ts`
الأداة: **OpenStreetMap Nominatim** — مجاني تماماً، بدون مفتاح API.

```ts
const url = `https://nominatim.openstreetmap.org/reverse?lat=${lat}&lon=${lng}&format=json&accept-language=ar`;
```

- بيرجّع `{ street, district }` بالعربي.
- **فيه كاش محلي** (Map) بدقة ~11 متر عشان مايكررش الطلبات لنفس المكان.
- لو فشل الطلب → يرجّع `{ street: "غير متاح", district: "غير متاح" }`.

**ملاحظة:** لو المستخدم كتب "الحي" يدوياً في الخانة القابلة للتعديل، ده **بيتغلّب**
على الحي اللي بييجي من الـ GPS.

## 6. تحويل الكلام للوحة — المعالج (أهم جزء)

الملف: `lib/plateParser.ts`
الدالة: `extractMultiplePlates(transcript: string): MultiPlateResult[]`

> كود محلي بالكامل — **لا AI، لا إنترنت**. مجرد regex + قواميس + خوارزمية.

### القاعدة الأساسية
كل لوحة سعودية = **1-3 حروف** + **4 أرقام** (تتكمّل بأصفار لو أقل).
الحروف الصحيحة فقط: `ا ب ح د ر س ص ط ع ق ك ل م ن هـ و ي`

### مراحل المعالجة (Pipeline)

```
1. تنظيف التشكيل (removeDiacritics) — يشيل الحركات فقط، مش الحروف
2. توحيد الألف: أ إ آ → ا
3. توحيد الهاء: ه → هـ
4. ى → ي
5. قاموس أسماء الحروف (LETTER_NAMES): دال→د، صاد→ص، لام→ل، سين→س ...
6. الدمج الصوتي (PHONETIC_MERGES): احلام→ا ح ل ...
7. الأرقام المنطوقة (SPOKEN_NUMBERS): خمسة→5، تلاتين→30، ألفين→2000 ...
8. توحيد الأرقام العربية: ٥→5
       ↓
9. تقطيع لـ "atoms" (ذرّات): كل كلمة تتصنّف:
   L = حرف لوحة | D = رقم | V = نوع سيارة | N = ملاحظة
       ↓
10. تجميع الأرقام في مجموعات (كل مجموعة 4 أرقام = لوحة واحدة)
       ↓
11. لكل مجموعة أرقام: امسح للخلف واجمع أقرب ≤3 حروف = لوحة
       ↓
12. باقي الكلمات → ملاحظات لأقرب لوحة، وأنواع السيارات → خانة النوع
```

### مثال حي (من تسجيل حقيقي)
```
تقول:            "دال لام لام تسعة ستة سبعة تسعة"
بعد التطبيع:      "د ل ل 9 6 7 9"
النتيجة:          دلل9679 ✅
```

### قواميس مهمة داخل الملف
- `LETTER_NAMES` — أسماء الحروف بالفصحى والعامية المصرية (دال، صاد، حاء، حا...).
- `EGYPTIAN_LETTERS` — النطق المصري القصير (حه→ح، دال→د...).
- `PHONETIC_MERGES` — كلمات بتتقال ملزوقة (احلام→ا ح ل).
- `SPOKEN_NUMBERS` — كل الأرقام المنطوقة من 0 لـ 9000 + المركّبة (خمسة وعشرين→25).
- `VEHICLE_TYPES` — ونيت، فان، دباب، شاحنة، باص، صالون، بيكاب، تاكسي، كروزر، باترول، نقليات، مفحوطة.
- `VALID_AR_LETTERS` — الـ 17 حرف المسموح بيهم على اللوحات السعودية.

### دالة اللوحة الواحدة
`parsePlateFromTranscript()` — نفس المنطق بس للوحة واحدة (بتستخدم كـ fallback).

### 🐛 باگات اتصلحت في المعالج
1. **regex التشكيل كان بياكل حروف اللوحات** — النطاق القديم `[ؐ-ًؚ-ٟ]` كان بيشمل
   حروف زي ح م ل ويمسحها! اتصلح لنطاق التشكيل الصحيح فقط.
2. **تفريغ اللوحات المتعددة كان بيتجاهل قاموس أسماء الحروف** — دلوقتي بيستخدم نفس
   الـ pipeline الكامل، فأسماء الحروف (لام، صاد، دال) بتتحوّل صح.

### الاختبارات
`__tests__/plateParser.test.ts` — **154 اختبار** يغطّوا كل الحالات
(لوحة واحدة، متعددة، أسماء حروف، ملاحظات، أنواع سيارات، أرقام منطوقة، حالات مشوّهة).

## 7. التخزين المحلي (IndexedDB)

الملف: `lib/idb.ts`

### شكل السجل الواحد (RecordingEntry)
```ts
interface RecordingEntry {
  localId: string;           // uuid محلي
  agentId: string;           // معرّف مستخدم Supabase
  plate: string;             // اللوحة بدون فراغات مثل أبح1234
  vehicleType?: string;      // ونيت / فان / دباب ...
  lat?: number;
  lng?: number;
  street?: string;
  district?: string;
  recordedAt: string;        // ISO timestamp
  audioBlobBase64?: string;  // (غير مستخدم حالياً بعد شيل تسجيل الصوت)
  mapsLink?: string;
  notes?: string;
  recorderName?: string;
  isManual?: boolean;        // true = إدخال يدوي، false = تسجيل صوتي
  synced?: boolean;          // اتزامن مع السيرفر؟
}
```

### الدوال
```ts
saveRecording(entry)         // يحفظ/يحدّث سجل
getAllRecordings(agentId)    // كل السجلات (مرتبة بالأحدث)
deleteRecording(localId)
updateGeodata(localId, street, district)
updateNotes(localId, notes)
```

## 8. تصدير الإكسيل

الملف: `lib/excel.ts` (بيستخدم مكتبة **xlsx / SheetJS**)

### ترتيب الأعمدة (من `buildRows` في صفحة التسجيل)
```
رقم اللوحة | نوع السيارة | الشارع | الحي | تاريخ التسجيل | GPS | ملاحظات
```
- التاريخ يتنسّق كـ `DD-MM-YYYY HH:mm` (مش ISO خام).
- خلايا الـ GPS بتبقى روابط خرايط قابلة للضغط.

### الدوال
```ts
buildExcelBlob(rows, sheetName)          // يبني ملف إكسيل في الذاكرة
openExcelBlob(blob, filename)            // يفتح الملف (موبايل: FileOpener، ويب: تنزيل)
shareExcelBlob(blob, filename, title)    // يشارك عبر قائمة المشاركة (واتساب...)
```

### اسم الملف
قابل للتعديل من خانة "اسم ملف الإكسيل"، افتراضياً `اكسيل-DD-MM-YYYY`.
القيمة بتتحفظ في `localStorage`.

## 10. لوحة التشخيص (Debug Panel)

في أسفل الصفحة لوحة قابلة للطي بتوريك مراحل التحويل:
- **0 — SR Status:** حالة محرك التعرف الصوتي.
- **1 — Raw Transcript:** النص الخام (اللي الجهاز سمعه) — **أهم حاجة للتشخيص**.
- **2 — Original Transcript:** النص الأصلي من المحرك.
- **3 — Normalized Transcript:** بعد التطبيع.
- **4 — Plate Parsing Result:** اللوحات المستخرجة.
- **5 — Vehicle Type Result:** أنواع السيارات.
- **6 — Notes Result:** الملاحظات المتبقية.

> لو التفريغ طلع غلط، افتح "1 — Raw Transcript" وابعته — بيوضّح إذا كانت المشكلة
> في السمع (المحرك) ولا في المعالجة (الكود).

## 12. الملخص التنفيذي (أهم النقاط)

1. **التفريغ محلي بالكامل** — مفيش AI أو سحابة، مجرد كود ذكي في `plateParser.ts`.
2. **الجهاز بيسمع ويحوّل لنص**، والكود المحلي بيحلل النص للوحات.
3. **الـ GPS** بيشتغل بمكتبة Capacitor على الموبايل، ونتيجته بتتحوّل لعنوان عربي
   عبر OpenStreetMap المجاني.
4. **التفريغ يدوي** بزر "ابدأ التفريغ" بعد المراجعة.
5. **عنق الزجاجة** = جودة سمع الجهاز للحروف المتهجّاة، مش الكود. النصيحة: انطق
   الحروف بأسماء واضحة وبوقفات.
6. **154 اختبار** بيحموا منطق التحويل من أي كسر مستقبلي.
